CHANGE 2 — Improve the Preview to render HTML email properly
════════════════════════════════════════════

The existing preview (showPreview) renders with dangerouslySetInnerHTML. Make it render in an iframe so HTML email styles don't leak into the SIGNAL UI, and so email-specific CSS (like table-based layouts from Designmodo) renders correctly.

Find the preview Dialog/panel (the one that uses dangerouslySetInnerHTML with DOMPurify.sanitize(body)).

Replace it with an iframe-based preview:

  {showPreview && (
    <Dialog open={showPreview} onOpenChange={setShowPreview}>
      <DialogContent className="max-w-2xl" style={{ height: '80vh', display: 'flex', flexDirection: 'column' }}>
        <DialogHeader>
          <DialogTitle>Email Preview</DialogTitle>
          <DialogDescription>
            Subject: {stepForm.subject || '(no subject)'}
          </DialogDescription>
        </DialogHeader>
        <div style={{ flex: 1, border: '0.5px solid var(--color-border-tertiary)', borderRadius: '8px', overflow: 'hidden' }}>
          <iframe
            title="Email preview"
            style={{ width: '100%', height: '100%', border: 'none', background: '#ffffff' }}
            srcDoc={`<!DOCTYPE html><html><head><meta charset="utf-8"><meta name="viewport" content="width=device-width, initial-scale=1.0"><style>body{margin:0;padding:0;background:#fff}</style></head><body>${stepForm.body || '<p style="color:#999;padding:20px;font-family:sans-serif;">No content yet</p>'}</body></html>`}

          />

          <Button variant="outline" onClick={() => setShowPreview(false)}>Close</Button>

      </DialogContent>
    </Dialog>
  )}
