Fix Gmail OAuth flickering in integrations-settings.tsx, client/src/hooks/useAuth.ts, and client/src/lib/queryClient.ts. Do NOT touch any other files.

--- FIX 1: client/src/hooks/useAuth.ts ---

Change refetchOnMount from true to false:
  refetchOnMount: false,

This prevents useAuth from firing a fresh network request every time the page 
navigates back after an OAuth redirect, which causes the loading flicker.

--- FIX 2: client/src/lib/queryClient.ts ---

In the getQueryFn function, find the block that handles res.status === 401:

  if (res.status === 401) {
    globalAuthFailed = true;
    setTimeout(() => {
      queryClient.invalidateQueries({ queryKey: ['/api/auth/user'] });
    }, 200);
  }

Replace the setTimeout invalidateQueries call with nothing — just set the flag 
and let useAuth's own staleTime handle re-checking naturally. The invalidateQueries 
inside setTimeout is what causes the flicker loop: it marks auth as stale, triggers 
a refetch, which causes isLoading to momentarily go true, which flashes the spinner.

New version:
  if (res.status === 401) {
    globalAuthFailed = true;
    // Don't immediately invalidate — let the staleTime expire naturally.
    // Aggressive invalidation here causes render flicker loops.
  }

In the GET /api/gmail-oauth/callback handler, improve the nonce-missing error
