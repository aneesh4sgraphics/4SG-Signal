Notion-Style UI Spec Sheet (Roboto, B&W, minimal)
1) Typography (Roboto only)

Font family: Roboto (400/500/700), no alternates.

Type scale (use these exact sizes):

Page Title: 24px / 32px, weight 600

Section Title: 16px / 24px, weight 600

Body: 14px / 20px, weight 400

Small/Meta: 12px / 16px, weight 400

Numbers (stats): 28px / 32px, weight 600

Rules

Never use more than 3 font weights (400, 500, 600/700).

Use sentence case everywhere (“Daily progress”, not “DAILY PROGRESS”).

Use muted text for supporting info, not smaller text.

2) Color system (pure grayscale)

Backgrounds

App background: #F7F7F7

Surface: #FFFFFF

Hover surface: #FAFAFA

Text

Primary: #111111

Secondary: #666666

Muted: #999999

Borders & dividers

Border: #EAEAEA

Divider: #EFEFEF

Focus ring

Focus outline: rgba(0,0,0,0.25) or #CFCFCF (no bright blue rings)

Status chips (only place you can use color)

Success dot: green (muted)

Warning dot: amber (muted)

Risk dot: red (muted)

Everything else stays grayscale.

3) Spacing + layout rhythm (this is the “Notion feel”)

Use an 8px grid. Only these spacings:

4, 8, 12, 16, 24, 32

Page padding

Desktop: p-6 outer padding

Section gaps: gap-6

Inside surfaces: p-4 or p-6 only

Max width

Content: max-w-6xl

Text-heavy areas: max-w-3xl

Rule

Prefer whitespace over boxes.

Use dividers more than “cards”.

4) Surfaces (cards, panels, tables)

Default surface

Background: white

Border: 1px #EAEAEA

Radius: 10–12px

Shadow: none or very subtle (Notion is mostly flat)

Don’t

No gradients

No thick outlines

No big shadows

No “glowing” CTA bars

5) Buttons (Notion-like)

You only need 3 button types:

Primary (rare)

Background: #111

Text: white

Radius: 10px

Height: 36px

Use only for the single most important action on a screen.

Secondary (default)

Background: white

Border: #EAEAEA

Text: #111

Hover: #FAFAFA

Ghost

Background: transparent

Text: #111

Hover: #FAFAFA

Rule

No colored buttons (except status actions if absolutely needed).

CTA overload kills Notion vibes.

6) Inputs + forms

Text input

Height: 36px

Border: #EAEAEA

Radius: 10px

Placeholder: #999

Dropdown

Same sizing as inputs

No fancy shadows

7) Tables (this is crucial for “Notion”)

Row height: 44px
Divider: 1px #EFEFEF
Hover row: #FAFAFA
Header row: small text 12px, muted (#666), no uppercase.

Tables should feel like “editable lists,” not enterprise software.

8) Chips / badges

Height: 22–24px

Font: 12px

Background: #F2F2F2

Border: #EAEAEA

Optional small colored dot for status only.

Page blueprints (so Replit knows what to build)
A) Dashboard (Notion AI vibe)

Top

“Today” header

Two tiny pills: Efficiency + 0/10 Today

Body: 2-column

Left: Now Mode (one card preview + “Continue”)

Right: “Queue” list (Overdue, Due soon, Needs data)

Below

Recent Activity (table/list)

Everything else in tabs (Tools / Pipeline / Admin)

Rule: No more than one screen of content before scrolling.

B) Client Detail Page

Sticky header (always visible):

Company name + tier chip

Rep assignment + contact quick actions (Email / Call)

3 key metrics (Spent, Orders, Engagement)

Two-column

Left: Next Best Move + Journey progress

Right: Category Trust + Machines + “Last touch”

Tabs below: Quotes / Orders / Emails / Samples / Press.

C) Now Mode

Make it look like a Notion task execution screen:

Top: progress bar + “10 remaining”

5 mini counters: Calls / Followups / Outreach / Hygiene / Enablement

Main: One task card

Customer name, email, phone

“Why now” (one sentence)

Outcome buttons (monochrome)

“Skip” is subtle (requires reason after repeated skips)

Add keyboard shortcuts:

1 = Connected

2 = Voicemail

3 = No Answer

4 = Bad Fit

S = Skip

Engineering instructions (so it actually stays consistent)

Tell Replit to implement:

Design tokens

styles/tokens.ts (colors, spacing, radii, type scale)

Base components

Button, Chip, Table, Stat, PageHeader, Section, Surface/Card

AppShell

Every page must render inside AppShell

No one-off styling

“If a component needs >8 Tailwind classes, it becomes a shared component.”

“Acceptance checklist” (use this to approve the work)

✅ Roboto everywhere

✅ Only grayscale UI (except small status dots)

✅ Flat surfaces, minimal borders, no gradients

✅ Tables look like Notion lists

✅ Dashboard/Client/Now Mode share the same spacing + typography

✅ Same buttons/inputs across the whole app
