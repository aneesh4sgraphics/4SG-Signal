In server/spotlightDigestWorker.ts, enhance the weekly digest email to include pipeline kanban summary.

Find the checkAndSendDigest function. Before it sends the email, add a kanban summary section by querying:

1. Count of leads with firstEmailReplyAt set (replied)
2. Count of leads with pressTestKitSentAt or sampleEnvelopeSentAt set (samples sent)  
3. Count of leads where lastContactAt < 10 days ago and firstEmailReplyAt is null (no response)
4. Count of leads with salesKanbanStage = 'issue'

Add these queries inside checkAndSendDigest before the email HTML is built:

  const tenDaysAgo = new Date();
  tenDaysAgo.setDate(tenDaysAgo.getDate() - 10);

  const [kanbanCounts] = await db.select({
    replied: sql<number>`COUNT(CASE WHEN first_email_reply_at IS NOT NULL THEN 1 END)::int`,
    samplesSent: sql<number>`COUNT(CASE WHEN press_test_kit_sent_at IS NOT NULL OR sample_envelope_sent_at IS NOT NULL THEN 1 END)::int`,
    noResponse: sql<number>`COUNT(CASE WHEN last_contact_at < ${tenDaysAgo} AND first_email_reply_at IS NULL AND first_email_sent_at IS NOT NULL THEN 1 END)::int`,
    issues: sql<number>`COUNT(CASE WHEN sales_kanban_stage = 'issue' THEN 1 END)::int`,
  }).from(leads);
