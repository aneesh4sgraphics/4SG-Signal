FIX 2 — Auto-calculate sqm from size in Product Mapping
════════════════════════════════════════════

In client/src/pages/product-mapping.tsx, find the mapping dialog/form 
where size is entered (the input for the size field like "12x18").

Add this helper function at the top of the component (not inside render):

  function calcSqmFromSize(sizeStr: string): number | null {
    const match = sizeStr.replace(/\s/g, '').match(/^(\d+\.?\d*)[xX×](\d+\.?\d*)/);
    if (!match) return null;
    const w = parseFloat(match[1]);
    const h = parseFloat(match[2]);
    if (!w || !h || w <= 0 || h <= 0) return null;
    return parseFloat(((w * h) / 1550.0031).toFixed(4));
  }

Wire it to the size input field's onChange handler:
When size changes, call calcSqmFromSize(newSize). If it returns a non-null 
value, automatically update the totalSqm field in the form state.

Show a small note below the sqm field when auto-calculated:
  <span style={{ fontSize: '11px', color: 'var(--color-text-tertiary)' }}>
    Auto-calculated from dimensions
  </span>

The user can still manually override the sqm value after auto-fill.

    return [
      p.dealerPrice, p.dealer2Price, p.exportPrice,
      p.masterDistributorPrice, p.retailPrice, p.approvalNeededPrice,
      p.tierStage25Price, p.tierStage2Price, p.tierStage15Price, p.tierStage1Price

  }
