PROBLEM 2 — Duplicate sizes shown for same product (e.g. GPOLY14-23x29B showing as 0.4303 sqm AND 43.0322 sqm)

The root cause: the same physical size has two entries in productPricingMaster — one with totalSqm representing a single sheet, and one representing a pack (e.g. 100 sheets). They may have different itemCodes (one with -ARCH suffix, one without).

In client/src/pages/quote-calculator.tsx, find the availableSizes deduplication logic (around line 550):
  const seen = new Set<string>();
  return filtered.filter(item => {
    const key = item.itemCode || item.size;
    if (seen.has(key)) return false;
    seen.add(key);
    return true;
  });

Change the deduplication key to use the physical size (width x height) extracted from the size field, so that two records with the same physical dimensions are treated as duplicates regardless of itemCode. When there are duplicates by physical size, keep the one with pricing data (i.e. dealerPrice > 0 or retailPrice > 0):

Replace the deduplication block with:
  // Deduplicate by physical size — keep the record that has pricing data
  const sizeMap = new Map<string, typeof filtered[0]>();
  for (const item of filtered) {
    // Normalize the size string as the key (e.g. "23x29", "23 x 29", "23X29" all → "23x29")
    const normalizedSize = (item.size || '').toLowerCase().replace(/\s+/g, '').replace(/x/g, 'x');
    const existing = sizeMap.get(normalizedSize);
    if (!existing) {
      sizeMap.set(normalizedSize, item);
    } else {
      // Prefer the record that has actual pricing data over $0 records
      const existingHasPrice = parseFloat(String(existing.dealerPrice || 0)) > 0 || parseFloat(String(existing.retailPrice || 0)) > 0;
      const itemHasPrice = parseFloat(String(item.dealerPrice || 0)) > 0 || parseFloat(String(item.retailPrice || 0)) > 0;
      if (!existingHasPrice && itemHasPrice) {
        sizeMap.set(normalizedSize, item);
      }
    }
  }
  return Array.from(sizeMap.values());
